# Image Prompt Template — Attributed Memory Vignette

A memory vignette is a subjective transformation of the bar, not an objective flashback and not a new explorable location.

---

## Required inputs

1. `[PATH_TO_APPROVED_BAR_COMPOSITION]` — controls the shapes that transform.
2. `[PATH_TO_SPEAKER_MASTER_OR_SILHOUETTE]` — controls who owns the memory.
3. `[PATH_TO_APPROVED_MEMORY_STYLE_TARGET]` — controls the limited project-specific secondary style.
4. `[PATH_TO_VERIFIED_REFERENCE_INDEX]` — supports material details; does not make the memory objectively true.

---

## Dramatic objective

Create one subjective memory tableau attributed to `[SPEAKER_ID]` about `[MEMORY_TOPIC]`.

The image should communicate `[EMOTIONAL_AND_NARRATIVE_BEAT]` while preserving uncertainty. It must contain one dominant symbolic object: `[SYMBOLIC_OBJECT]`.

Transform existing bar geometry:

- `[BAR_SHAPE_1]` becomes `[MEMORY_SHAPE_1]`;
- `[BAR_SHAPE_2]` becomes `[MEMORY_SHAPE_2]`;
- `[BAR_SHAPE_3]` becomes `[MEMORY_SHAPE_3]`.

The present-day speaker remains visible as `[SILHOUETTE/EDGE/REFLECTION DESCRIPTION]`, anchoring the scene as testimony.

---

## Style

- 4–7 controlled colors;
- screen-print, cut-paper, or painted-paper construction;
- visible layering and restrained overprint misregistration;
- simplified, legible bodies;
- one strong value hierarchy;
- no photorealism, cinematic flashback grading, dream-glow cliché, or unrelated surrealism;
- no claim that every visual detail is literal truth.

Use the approved memory palette `[PALETTE]` and preserve the main game's silhouette language.

---

## Scope limits

- one tableau or small set of transformable layers;
- 30–90 seconds in engine;
- no crowd of distinct faces;
- no new full background requiring independent exploration;
- no generated text, numbers, banners, labels, insignia, or handwriting;
- text/numbers required by the story must be runtime vector/text layers;
- do not reproduce archival photos or real identifiable victims;
- no gore or sensational staging.

---

## Contradiction support

Design `[1–3]` swappable layers so another character's contradiction can alter the memory without regenerating the whole scene, for example:

- figure count;
- uniform/coat silhouette class;
- position of a document or machine gauge;
- weather/exhaust/smoke layer;
- whether the speaker is central or peripheral.

Record which elements are speaker assertion, verified context, and deliberate uncertainty.

---

## Codex QA

Verify that the vignette:

- clearly derives from the bar composition;
- remains attributed to the speaker;
- is readable within three seconds;
- contains no generated lettering;
- does not create a factual claim absent from the ledger;
- has separable contradiction layers;
- returns cleanly to the present room;
- does not stylistically contaminate the main environment or character masters.
